Status code and Success message:

  • Успешный POST запрос - код ответа: 201, сообщение ответа: "POST request successful. Created"
  • Успешный GET запрос - код ответа: 200, сообщение ответа: "GET request successful"
  • Успешный GET запрос по ID - код ответа: 200, сообщение ответа: "GET_ID request successful. OK"
  • Успешный GET запрос по ID, но на сервере ресурс с таким ID отсутствует - код ответа: 404, сообщение ответа: "Not Found"
  • Успешный PUT запрос - код ответа: 204, сообщение ответа: "PUT request successful. No Data"
  • Успешный DELETE запрос - код ответа: 204, сообщение ответа: "DELETE request successful. No Data"
  • Успешный OPTIONS запрос - код ответа: 200, сообщение ответа: "OPTIONS request successful. OK"

Status code and Error message:

  • Внутренняя ошибка сервера - код ответа: 500, сообщение ответа: "Server error"
  • Ошибка сервера при попытки подключения к базе данных - код ответа: 503, сообщение ответа: "error_db_connection"
  • Ошибка сервера при запросе к базе данных. Запрос к серверу отправлен методом POST - код ответа: 502, сообщение ответа: "POST method error"
  • Ошибка сервера при запросе к базе данных. Запрос к серверу отправлен методом GET - код ответа: 502, сообщение ответа: "GET method error"
  • Ошибка сервера при запросе к базе данных. Запрос к серверу отправлен методом PUT - код ответа: 502, сообщение ответа: "PUT method error"
  • Ошибка сервера при запросе к базе данных. Запрос к серверу отправлен методом DELETE - код ответа: 502, сообщение ответа: "DELETE method error"
  • Ошибка сервера при запросе к базе данных. Запрос к серверу отправлен методом GET с ID - код ответа: 502, сообщение ответа: "GET_ID method error"

GET

Получаем все алгоритмы с записями из базы данных в формате JSON
Запрос отправляется методом GET без параметров и с пустым телом запроса.
Возращаемые данные с сервера 
массив с данными =
[
    {
        id: идентификатор алгоритма,
        name: имя алгоритма
    },
    {}...
]

GET с параметрами ID

Получаем конкретный алгоритм по его ID из базы данных в формате JSON
Запрос отправляется методом GET c параметром ID алгоритма и с пустым телом запроса.
Возращаемые данные с сервера 
{
    id: идентификатор алгоритма,
    name: имя алгоритма
    records : массив с записями
}
массив с записями =
[
    {
        done: false или true - выполнена ли запись,
        text: текст одной записи
    },
    {}...
]

POST

Добавляем в базу данных новый алгоритм
Запрос отправляется методом POST и с заполненным телом запроса. 
{
     id: "",
     name: имя алгоритма,
     records : массив с записями
}
массив с записями =
[
    {
        done: false или true - выполнена ли запись,
        text: текст одной записи
    },
    {}...
]
Ответ, возвращаемый с сервера не имеет тела, но при этом имеет заголовок Location, в котором указа URL новосозданного ресурса (≪ имя сервера≫/?id=≪id нового алгоритма≫)

PUT

Обновляем уже существующий алгоритм в базе данных
Запрос отправляется методом PUT и с заполненным телом запроса. 
{
     id: "",
     name: имя алгоритма,
     records : массив с записями
}
массив с записями =
[
    {
        done: false или true - выполнена ли запись,
        text: текст одной записи
    },
    {}...
]
Ответ, возвращаемый с сервера не имеет тела.

DELETE

Удаляет из базы данных существующий алгоритм
Запрос отправляется методом DELETE c параметром ID алгоритма и с пустым телом запроса.
Ответ, возвращаемый с сервера не имеет тела.

OPTIONS

Получает список методов, доступных по данному URI.
Запрос отправляется методом OPTIONS без параметров и с пустым телом запроса.
Ответ, возвращаемый с сервера не имеет тела, но при этом имеет заголовок Allow, в котором перечислены методы, которыми можно обращаться к серверу.
Ваш алгоритм успешно сохранен
